# Architecture Decision Records (ADRs)

This directory contains Architecture Decision Records documenting important architectural decisions.

## Format

Each ADR should follow this format:

```
# ADR-XXXX: Title

**Date:** YYYY-MM-DD
**Status:** Proposed | Accepted | Deprecated | Superseded

## Context
[Describe the context and problem]

## Decision
[Describe the decision made]

## Consequences
[Describe the consequences, both positive and negative]

## Alternatives Considered
[List alternatives that were considered]
```

## Examples

- `0001-use-clean-architecture.md`
- `0002-choose-postgresql.md`
- `0003-two-frontend-implementations.md`
